MacroTraits.jl

MacroTraits.jl implements a small trait-dispatch surface with four macros:

  1. @def_trait defines a trait and its built-in mappings.
  2. @trait_dispatch defines the public entry point.
  3. @trait_function defines trait-state-specific implementations.
  4. @trait_map extends an existing trait with new mappings.

The package is intentionally narrow. It routes on the trait of the first argument, preserves the public signature you wrote, and keeps the generated worker functions private.

Quick Start

@def_trait Sequence begin
    DynamicVector => [Vector]
    StaticTuple   => [Tuple, NamedTuple]
end

@trait_dispatch process_items(x) :: Sequence

@trait_function process_items(x) :: DynamicVector = :dynamic
@trait_function process_items(x) :: StaticTuple = :static

process_items([1, 2, 3])
# :dynamic

process_items((1, 2, 3))
# :static

Operational Model

At runtime, the public function computes TraitName(first_argument) and forwards the trait-state value plus the original arguments to a hidden worker generated by MacroTraits.

If trait resolution succeeds but no matching @trait_function implementation exists, the call fails as the public function rather than surfacing the hidden worker in the normal error path.

Signature Rules

MacroTraits intentionally supports only simple positional signatures written as name or name::Type.

  • Supported:
    • f(x)
    • f(x, y)
    • f(x::Vector{Int})
  • Rejected during macro expansion:
    • default values
    • keyword arguments
    • varargs
    • destructuring arguments

This narrow surface keeps the generated methods predictable and helps preserve clear error behavior.

Extension Guidance

Prefer @trait_map when extending an existing trait with new types. It emits the same foldable, constprop trait-resolution methods as @def_trait.

For manual trait-resolution methods, return a trait-state instance such as StaticTuple(), and treat MacroTraits worker functions as internal implementation details.

Documentation Placement

Attach public API docstrings to @trait_dispatch. Attach implementation-specific docstrings to @trait_function methods.
